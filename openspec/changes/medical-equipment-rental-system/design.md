## Context

本课程设计项目为医疗器械租赁管理系统，面向医疗器械租赁公司，需要管理设备信息、客户信息、租赁订单、费用结算等核心业务。项目从零搭建，采用 Next.js 全栈架构。

### 技术栈选择
- **框架**：Next.js 14 (App Router) — 全栈框架，前端 React + 后端 API Routes 合一
- **前端 UI**：React 18 + TypeScript + Ant Design 5 + Tailwind CSS
- **后端**：Next.js API Routes + Prisma ORM + MySQL 8.0
- **认证**：NextAuth.js (JWT Session)
- **图表**：ECharts / Recharts
- **构建工具**：pnpm

### 角色设计

| 角色 | 权限职责 | 主要操作 |
|------|---------|---------|
| 招商业务员 | 客户开发、租赁业务接洽 | 创建租赁订单、查看订单进度、维护客户信息 |
| 库房人员 | 设备实物管理 | 设备入库/出库/盘点、设备状态维护、出入库记录 |
| 运维工程师 | 设备技术维护 | 设备维修记录、保养计划、设备报废鉴定 |
| 财务 | 费用核算 | 租金计算、押金管理、费用结算、收入统计 |

## Goals / Non-Goals

**Goals:**
- 构建完整可运行的医疗器械租赁管理系统全栈项目
- 实现四种角色的认证与权限控制（招商业务员/库房人员/运维工程师/财务）
- 实现设备、客户、租赁订单、费用结算 CRUD 全流程
- 多角色协同完成租赁业务闭环
- 代码结构清晰，便于课设答辩展示和 React / Next.js 学习
- 符合课设要求的文档完整性

**Non-Goals:**
- 不涉及真实支付对接（费用为系统内记账）
- 不涉及硬件设备对接（如医疗设备物联网）
- 不涉及高并发、分布式等企业级特性
- 不涉及移动端适配
- 不涉及消息队列等中间件

## Decisions

### 1. Next.js 全栈 vs 前后端分离
- **选择**：Next.js App Router 全栈架构
- **理由**：一个项目同时包含前端页面和后端 API，降低项目搭建复杂度；配合 Prisma 可快速完成全栈 CRUD；作为课设可以展示对现代全栈框架的理解；你想借机学 React，Next.js 是最佳实践起点

### 2. ORM 框架选择
- **选择**：Prisma
- **理由**：TypeScript 原生支持，类型安全，Schema 驱动开发（定义模型自动生成类型），迁移工具链完善，适合课设快速开发

### 3. 前端 UI 库
- **选择**：Ant Design 5 + Tailwind CSS
- **理由**：Ant Design 提供丰富的企业级管理后台组件（表格、表单、弹窗、布局），开箱即用减少样式工作量；Tailwind CSS 辅助处理自定义样式

### 4. 权限模型
- **选择**：基于角色的 RBAC（4 种角色），后端 Middleware 校验 + 前端路由守卫
- **理由**：四种角色的业务流程明确区分，基于角色控制页面和 API 访问权限能满足需求；前端根据角色动态渲染菜单和操作按钮

### 5. 认证方案
- **选择**：NextAuth.js 配合 JWT 凭证登录
- **理由**：Next.js 生态最成熟的认证库，支持 Credentials Provider（账号密码）、JWT Session、Middleware 路由保护

### 6. 数据库设计
- **选择**：Prisma Schema 驱动（Schema → Migration → 数据库）
- **理由**：Prisma 自动生成 Migration 脚本，无需手写 SQL；模型定义清晰，便于后期修改和维护

## Risks / Trade-offs

- **Next.js 学习曲线** → 你正好想学 React / Next.js，我会在实现过程中注释关键代码，逐步推进
- **前后端耦合在同一项目** → 通过清晰的目录结构（`src/app/api/` 和 `src/app/(pages)/`）分离关注点
- **课设时间紧张** → 优先核心业务流程（租赁订单 CRUD），统计报表等非核心功能放在最后
- **Ant Design 定制灵活性** → 管理系统场景足够，不需要大量自定义样式
