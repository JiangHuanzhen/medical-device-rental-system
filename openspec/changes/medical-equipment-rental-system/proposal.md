## Why

作为一名软件工程专业大三学生，本课程设计旨在开发一个医疗器械租赁管理系统，解决传统租赁管理方式效率低、信息不透明、记录易丢失等问题。通过系统化管理医疗设备的入库、租赁、归还及费用结算流程，提升租赁业务的运作效率和数据可追溯性。同时借助此次课设学习 React / Next.js 全栈开发技术。

## What Changes

- **用户管理**：支持四种角色（招商业务员、库房人员、运维工程师、财务）登录与权限控制
- **设备管理**：医疗器械的增删改查，包括设备分类、规格、状态等信息维护；库房人员负责设备入库出库
- **客户管理**：租赁客户（医疗机构）信息的登记与管理
- **租赁管理**：租赁订单的创建、审核、归还、续租等全流程管理；招商业务员发起，多角色协同
- **费用管理**：租金计算、押金管理、费用结算与记录；财务角色负责结算
- **数据统计**：租赁记录统计、设备利用率、财务收入等看板
- **系统基础架构**：从零搭建 Next.js + Prisma + MySQL 全栈项目骨架

## Capabilities

### New Capabilities
- `user-auth`: 用户登录、角色权限管理（招商业务员/库房人员/运维工程师/财务）
- `equipment`: 医疗器械基础信息管理（分类、品牌、型号、状态、出入库）
- `customer`: 客户（医疗机构）信息管理
- `rental-order`: 租赁订单的创建、审核、归还、续租全流程
- `billing`: 租金计算、押金与费用结算（财务角色专属）
- `dashboard`: 数据统计看板与报表
- `system-config`: 系统基础配置（字典、参数等）

### Modified Capabilities
无（初始项目，尚无已有规范）

## Impact

- 新建 Next.js 全栈项目（React + TypeScript + API Routes）
- 使用 Prisma ORM 连接 MySQL 数据库
- 使用 Ant Design React UI 组件库快速搭建管理界面
- 影响范围：全栈项目搭建，涉及所有模块
