# Stakeholder Chat — 涉众智能体对话

通过 API 调用老师平台的涉众智能体，获取需求。支持四个角色：招商业务员、库房人员、运维工程师、财务。

## 用法

```
/stakeholder <角色> <问题>
```

### 示例

```
/stakeholder 招商业务员 请介绍一下你日常工作流程
/stakeholder 库房人员 设备入库时遇到什么困难
/stakeholder 运维工程师 故障报修流程是怎样的
/stakeholder 财务 租金计算和结算怎么处理
```

## 参数

| 参数 | 必填 | 说明 |
|------|------|------|
| 角色 | 是 | 招商业务员 / 库房人员 / 运维工程师 / 财务 |
| 问题 | 是 | 你想问涉众的问题 |

## 实现方式

调用 `POST https://210.34.148.101:5000/api/chat` 接口：

```json
{
  "project_id": "10",
  "stakeholder": "<角色>",
  "phase": "v1",
  "question": "<问题>"
}
```

响应格式：

```json
{
  "answer": "<涉众的回答>"
}
```

## 注意事项

- **phase 参数始终为 `v1`**（老师的平台当前只有 v1 阶段）
- **project_id 固定为 `10`**（当前项目的 ID）
- 如果涉众的回答中包含需要追问的内容，可以继续用 `/stakeholder` 追问
- 所有对话记录会由老师的平台自动保存到知识库

## 推荐提问策略

为获取全面需求，建议按以下顺序和每个涉众进行多轮对话：

### 第1轮：场景引导（问日常工作）
> 请描述你一天的工作流程是怎样的？

### 第2轮：痛点挖掘（问麻烦的地方）
> 目前最让你头疼的问题是什么？

### 第3轮：异常路径（问边界情况）
> 如果某个环节出了问题，你们怎么处理？

### 第4轮：数据边界（追问细节）
> 比如数量、频率、周期、金额范围等具体数字
