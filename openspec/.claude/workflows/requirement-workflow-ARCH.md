
# 需求工程 LangGraph 工作流

```

                              │                                  ▼
    A1_招商业务员 ──→    A1_汇总    ──→    A2_需求分析    ──→    A3_UML建模
    A1_库房人员   ──→      ↑              │                      │
    A1_运维工程师  ──→      │              │  ← 严重问题退回      │
    A1_财务      ──→       │              ▼                      │
                              │          ┌──────────┐            │
                              │          │ A4_SRS   │ ←──────────┘
                              │          │ 生成     │
                              │          └────┬─────┘
                              │               │
                              │               ▼
                              │          ┌──────────┐

                              │          └────┬─────┘

                              │

```

## 工作流说明

每条边的含义：
- **A1(4个涉众) → A1_汇总**：四个涉众对话完成后，汇总需求
- **A1_汇总 → A2_需求分析**：进入质量检测
- **A2_需求分析 → (条件)**：
  - 有严重问题 → 回退A1重新获取
  - 没问题 → 进入A3建模
- **A3_建模 → A4_SRS生成**：UML模型作为SRS输入
- **A4_SRS → A5_验证**：交叉验证
- **A5_验证 → (条件)**：
  - 获取类问题 → 退回A1
  - 分析类问题 → 退回A2
  - 通过 → 进入CCB
- **CCB → (条件)**：人工审批决定
- **CCB通过 → A6_基线创立**：归档冻结
