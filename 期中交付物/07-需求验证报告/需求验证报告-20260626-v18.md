## 发现清单（教材§7四类交叉比对）

|-----|---------|---------|---------|---------|
| 历史遗漏 | 重要 | 3.1.2 设备管理模块 (EQP) | SRS遗漏了需求清单中BR-EQP-010（库房人员全面复核）的详细描述。SRS中FR-EQP-004的第4步提到了库房人员复核，但描述为“库房人员必须对所有检测项（而非仅系统标记的可疑项）进行逐项复核”，这与BR-EQP-010的涉众原话“所有归还设备的初检结果都必须经过库房人员全面复核，而不仅仅是系统标记的可疑项”在语义上一致，但SRS未引用该需求条目，也未在关联需求条目中列出BR-EQP-010。 | 在FR-EQP-004的“关联需求条目”中补充BR-EQP-010, UR-EQP-010。 |
| 对话偏差 | 重要 | 3.1.2 FR-EQP-003 | SRS中FR-EQP-003的紧急出库审批流程描述为“区域经理 -> 质量合规负责人 -> 招商业务总监”三级审批。但涉众原话（BR-EQP-003）中明确要求“必须由区域经理+质量合规负责人+招商业务总监三级审批”，并未指定顺序。SRS强制规定了审批顺序，属于对涉众意图的局部偏离。 | 将审批流程描述修改为“由区域经理、质量合规负责人、招商业务总监依次或并行审批”，或与涉众确认是否需要指定顺序。 |
| 对话偏差 | 重要 | 3.1.2 FR-EQP-004 | SRS中FR-EQP-004的偏差处理逻辑（第3步）描述为：偏差超过预警阈值但未超过干预阈值时，系统弹出弹窗，提供“强制入库”和“创建维修工单”两个选项。但涉众原话（BR-EQP-009）中，运维工程师建议“优先采用预警弹框，提供‘强制入库’和‘创建维修工单’两个可选操作”，并强调这是“更为合理”的方案。SRS将此作为唯一处理方式，未体现涉众对“优先采用”的倾向性，且未提及涉众原话中“传感器偶发漂移、环境干扰”等导致偏差的合理场景。 | 在SRS中补充说明：该弹窗是默认处理方式，并提及涉众原话中关于偏差合理性的背景，或增加配置项允许运维工程师调整此行为。 |
| 对话偏差 | 建议 | 3.1.2 FR-EQP-004 | SRS中FR-EQP-004的偏差处理逻辑（第3步）描述为：偏差超过干预阈值时，系统自动执行“锁定设备”、“终结当前检测流程”、“自动创建P1级维修工单”的动作链。涉众原话（BR-EQP-008）中，运维工程师明确要求“流程终结并触发新工单”，并详细描述了“三步动作链”。SRS的描述与涉众原话基本一致，但SRS未提及“设备资产状态立即变为‘待维修-锁定’”这一具体状态名称，而是使用了“待维修-锁定”的表述，与涉众原话中的“待维修-锁定”一致，但SRS在数据字典中未定义此状态。 | 在数据字典的device表status字段中补充枚举值“待维修-锁定”。 |
| 对话偏差 | 建议 | 3.1.2 FR-EQP-004 | SRS中FR-EQP-004的“全面复核”步骤描述为“库房人员必须对所有检测项（而非仅系统标记的可疑项）进行逐项复核”。涉众原话（BR-EQP-010）中，库房人员明确要求“所有归还设备的初检结果都必须经过库房人员全面复核，而不仅仅是系统标记的可疑项”。SRS的描述与涉众原话一致，但SRS未提及涉众原话中关于“责任闭环与风险管控”、“设备状态与存放位置确认”等复核目的。 | 在SRS中补充复核的目的和意义，以增强需求的可理解性。 |
| 文档矛盾 | 重要 | 3.1.2 FR-EQP-001 与 2.6 假设与依赖 | FR-EQP-001的验收标准中，要求“创建一个校准有效期为7天后的设备，状态为‘库房待租’。系统在次日凌晨扫描后，该设备应被标记为‘预警中’”。但2.6节假设与依赖中，DEP-01指出“设备出厂基准数据已录入系统”，而FR-EQP-001的预警规则依赖于“校准有效期”字段，与出厂基准数据无关。这并非直接矛盾，但验收标准中使用了“校准有效期”而非“出厂基准数据”，与DEP-01的表述不一致，可能引起混淆。 | 将DEP-01的表述修改为“设备校准有效期信息已录入系统”，或明确区分“出厂基准数据”和“校准有效期”两个概念。 |
| 文档矛盾 | 建议 | 3.1.2 FR-EQP-002 与 3.1.7 FR-CFG-001 | FR-EQP-002中，出库管控的规则是“校准有效期剩余天数 > 7天，允许出库；<= 7天 且 > 3天，禁止出库；<= 3天，强制锁库”。FR-CFG-001中，系统管理员可配置“禁止出库天数”和“锁库天数”，默认值分别为7和3。但FR-EQP-002的规则是硬编码的，而FR-CFG-001允许动态配置。如果管理员修改了配置值，FR-EQP-002的规则将不再适用，导致内部不一致。 | 修改FR-EQP-002，使其引用FR-CFG-001中配置的阈值，而非硬编码数值。例如，将规则描述改为“若设备校准有效期剩余天数 > 系统配置的‘禁止出库天数’，允许出库；若剩余天数 <= ‘禁止出库天数’ 且 > ‘锁库天数’，禁止出库；若剩余天数 <= ‘锁库天数’，强制锁库”。 |
| 内部不一致 | 建议 | 3.1.2 FR-EQP-001 与 3.3.1 NFR-NFR-PERF-005 | FR-EQP-001中，校准预警扫描任务在“每日凌晨02:00”执行。NFR-NFR-PERF-005中，要求“每日凌晨的校准预警扫描任务，必须在30分钟内完成”。两者在时间上一致，但FR-EQP-001未明确提及性能要求，而NFR中提出了具体指标。这并非矛盾，但建议在FR-EQP-001中引用该性能需求，以增强可追溯性。 | 在FR-EQP-001的“后置状态”或“验收标准”中，增加对性能要求的引用。 |
