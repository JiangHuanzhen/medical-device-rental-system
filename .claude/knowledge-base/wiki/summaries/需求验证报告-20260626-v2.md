# 需求验证报告

验证日期：20260626
总体结论：**获取类问题**
发现 7 个问题

## 发现清单（教材§7四类交叉比对）
| 类型 | 严重程度 | 涉及章节 | 问题描述 | 修正建议 |
|-----|---------|---------|---------|---------|
| 历史遗漏 | 重要 | 3.1.2 设备管理 | SRS中缺少对设备归还后专业功能测试（如传感器灵敏度、电池容量）结果记录与出厂参数比对的功能需求。需求清单中BR-EQP-009明确提出了此需求。 | 在SRS 3.1.2节中新增功能需求，描述系统如何支持记录归还检测中的专业测试数据（传感器灵敏度、电池容量），并实现与出厂参数的自动比对及阈值告警。 |
| 对话偏差 | 重要 | 3.1.2 设备管理 / FR-EQP-003 | SRS中紧急出库审批流程（FR-EQP-003）的业务规则描述为“三方会签顺序固定：科室主任 -> 库房主管 -> 质控部门”。然而，需求清单中BR-EQP-007的涉众原话为“将系统里的紧急出库签字人修改为‘科室主任、库房主管、质控部门’三方”，并未明确指定审批顺序。SRS添加了顺序约束，属于“合理诠释”，但此顺序可能影响业务灵活性。 | 建议回A2确认审批顺序是否为硬性要求。如果业务上允许并行审批或顺序可配置，则应修改SRS以反映此灵活性。 |
| 对话偏差 | 重要 | 3.1.2 设备管理 / FR-EQP-006 | SRS中设备校准有效期预警（FR-EQP-006）的业务规则描述为“预警范围覆盖所有状态的设备，包括‘已出库’”。而需求清单中BR-EQP-004的涉众原话为“预警范围：所有设备，包括已出租的”。SRS将“已出租”具体化为“已出库”，但“已出库”是系统内部状态，而“已出租”可能包含更广泛的含义（如已出库但未归还）。此处的语义等价判断存在偏差。 | 建议回A2确认“已出租”在业务语境下的确切含义，是否等同于系统状态“已出库”，还是指代其他状态。SRS应使用与业务一致的术语。 |
| 对话偏差 | 建议 | 3.1.2 设备管理 / FR-EQP-006 | SRS中设备校准有效期预警（FR-EQP-006）的业务规则描述为“定时推送频率为每周一次（周一）”。而需求清单中BR-EQP-005的涉众原话为“每周一上午推送一次临期设备清单”。SRS的描述与涉众原话基本一致，但未提及“上午”这一具体时间点，属于轻微偏差。 | 建议在SRS中补充“上午”这一时间约束，以更精确地匹配涉众需求。 |
| 对话偏差 | 建议 | 3.1.2 设备管理 / FR-EQP-007 | SRS中设备出库前检测复核（FR-EQP-007）的业务规则描述为“非关键项的复核比例可在系统配置中设置，默认为30%”。而需求清单中BR-EQP-011的涉众原话为“30%的抽检比例不足以覆盖风险”，并未同意或建议30%作为默认值。SRS将30%设为默认值，与涉众意图存在偏差。 | 建议回A2确认非关键项的默认复核比例。如果涉众认为30%不足，应调整默认值或明确说明该值可由系统管理员配置。 |
| 文档矛盾 | 重要 | 3.1.2 设备管理 / FR-EQP-001 | SRS中设备入库校验（FR-EQP-001）的关联需求条目写为“BR-EQP-006 / UR-EQP-006”。然而，需求清单中BR-EQP-006的描述是“建立设备档案信息录入的自动校验机制”，与设备入库校验功能相符。但是，需求清单中BR-EQP-006的原始需求ID是UR-EQP-006，而SRS中FR-EQP-001的关联需求条目也引用了UR-EQP-006。此处的引用关系正确，但SRS中FR-EQP-001的业务流程描述为“系统读取对应的采购订单信息”，而需求清单中BR-EQP-006的涉众原话是“入库时，系统会强制要求比对采购订单与实物信息”。SRS的描述与涉众原话一致，但SRS中“关联需求条目”字段的写法（BR-EQP-006 / UR-EQP-006）与SRS文档头部“量化业务目标”表格中的写法（BR-EQP-006 / UR-EQP-006）一致，但SRS中“关联需求条目”字段的格式不统一。例如FR-EQP-002的关联需求条目是“BR-EQP-001 / UR-EQP-001”，而FR-EQP-003的关联需求条目是“BR-EQP-002, BR-EQP-007 / UR-EQP-002, UR-EQP-007”。这种格式不一致属于文档矛盾。 | 统一SRS中所有“关联需求条目”字段的书写格式，建议使用逗号分隔多个条目，如“BR-EQP-001, BR-EQP-002 / UR-EQP-001, UR-EQP-002”。 |
| 内部不一致 | 建议 | 3.1.2 设备管理 / FR-EQP-005 | SRS中设备归还检测（FR-EQP-005）的业务规则描述为“归还检测模板必须与入库检测模板完全一致”。然而，SRS中并未定义“入库检测模板”的具体内容或ID，导致无法验证“完全一致”这一约束。这属于SRS内部定义不一致。 | 建议在SRS中明确“入库检测模板”的定义，或引用一个具体的模板ID，以确保“完全一致”的约束是可验证的。 |
