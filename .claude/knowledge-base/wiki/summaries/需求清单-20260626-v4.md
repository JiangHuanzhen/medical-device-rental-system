| 业务目标描述 | 实现设备归还后状态的自动判定，减少人工操作和主观判断。 |
| 原始需求ID | UR-EQP-080 |
| 需求来源 | raw/notes/运维工程师-20260626-1326-需求记录.md（运维工程师） |
| 需求陈述 | 「如果检查记录中有“其他异常”或“需人工复核”字段，运维工程师可以根据现场判断手动选择开启“需维修”标记。但该手动操作会触发补充记录（如备注原因），并关联到后续维修流程。」 |
| 需求提炼 | 作为运维工程师，我希望系统在自动判定之外，支持我根据现场判断手动将设备标记为“需维修”，并强制要求填写原因备注，以便处理自动检测无法覆盖的异常情况，并确保流程可追溯。 |
| 优先级 | 中 |
