# 需求验证报告

验证日期：20260626
总体结论：**获取类问题**
发现 8 个问题

## 发现清单（教材§7四类交叉比对）
| 类型 | 严重程度 | 涉及章节 | 问题描述 | 修正建议 |
|-----|---------|---------|---------|---------|
| 历史遗漏 | 阻塞性 | §3.1.2 设备管理模块 | SRS遗漏了需求清单中BR-EQP-008（已出租设备定义）对应的功能需求。该需求明确了“已出租”设备的定义范围，直接影响校准预警（FR-EQP-003）等功能的准确性。SRS中未定义此概念，可能导致功能实现与涉众意图不一致。 | 在SRS中增加一条功能需求（如FR-EQP-006），明确定义“已出租”设备的状态转换条件，并关联到校准预警等功能的触发逻辑。 |
| 对话偏差 | 重要 | §3.1.2 FR-EQP-003 校准有效期预警 | SRS描述预警推送策略为“每周一凌晨02:00自动触发”和“每周一上午09:00推送”。但涉众原话（BR-EQP-004）要求的是“按周推送临期设备清单，并在到期前30天、15天、7天单独提醒”。SRS将“到期前30天、15天、7天”的提醒描述为“关键节点提醒”，但未明确说明这些提醒是针对“到期前”的日期，而非“到期日”当天。这可能导致实现偏差。 | 修改FR-EQP-003的业务流程描述，明确“关键节点提醒”是在设备校准有效期到期前的30天、15天、7天触发，而非到期日当天。例如：'对于校准有效期在未来30天、15天、7天内的设备，在对应日期的上午09:00单独发送提醒。' |
| 对话偏差 | 重要 | §3.1.2 FR-EQP-004 设备归还检测 | SRS描述“非关键项抽检”的默认比例为50%，但涉众原话（BR-EQP-009）中运维工程师建议的默认比例是50%，而SRS将此作为固定默认值。涉众原话还提到“支持管理员根据设备类型、供应商质量等条件动态调整”，SRS虽在业务规则中提及，但在验收标准中未体现此动态调整能力，可能导致实现时忽略此灵活性。 | 在FR-EQP-004的验收标准中增加一条：'系统管理员应能通过FR-CFG-002配置不同设备类型或供应商的非关键项抽检比例，且配置后立即生效。' |
| 对话偏差 | 重要 | §3.1.2 FR-EQP-002 紧急出库审批 | SRS描述紧急出库审批流程为“库房主管 → 质控部门 → 科室主任”，但涉众原话（BR-EQP-007）中库房人员明确要求“强制”此顺序。SRS虽在业务规则中提及“审批顺序必须严格遵循”，但在验收标准中未明确验证“强制”性，即系统是否阻止用户跳过或修改顺序。 | 在FR-EQP-002的验收标准中增加一条：'尝试以非指定顺序（如先推送给科室主任）提交审批，系统应拒绝并提示必须按指定顺序进行。' |
| 文档矛盾 | 重要 | §3.1.2 FR-EQP-001 与 §3.1.2 FR-EQP-002 | FR-EQP-001描述“校验逻辑适用于所有状态的设备，包括‘已出租’设备（在归还时触发）”。但FR-EQP-002的紧急出库审批是针对“校准过期”的设备。如果已出租设备校准过期，按照FR-EQP-001的逻辑，归还时触发校验并禁止操作，但FR-EQP-002的紧急出库审批流程是针对“出库”场景设计的，未明确说明是否适用于“归还”场景。这可能导致已出租设备归还时校准过期，无法走紧急审批流程。 | 明确FR-EQP-002的适用范围。如果紧急出库审批仅适用于出库场景，则需在FR-EQP-001中为“已出租设备归还”场景增加单独的异常处理流程（如“紧急归还审批”）。或者，扩展FR-EQP-002使其同时适用于出库和归还场景。 |
| 文档矛盾 | 建议 | §3.1.2 FR-EQP-004 与 §3.1.2 FR-EQP-005 | FR-EQP-004（设备归还检测）和FR-EQP-005（设备出库前检测）的业务流程描述高度相似，均包含“自动比对”、“关键项100%复核”、“非关键项50%抽检”。但SRS未明确说明这两个检测清单是否可以不同（由管理员配置）。FR-CFG-002提到“可为不同设备类型创建或编辑标准化检查清单”，但未明确区分“归还检测清单”和“出库前检测清单”。 | 在FR-CFG-002中明确区分“归还检测清单”和“出库前检测清单”的配置，并确保FR-EQP-004和FR-EQP-005的业务流程描述与配置能力一致。 |
| 内部不一致 | 建议 | §3.1.2 FR-EQP-003 与 §3.4 数据字典 | FR-EQP-003描述预警范围“必须覆盖所有状态的设备”，包括“已出租”。但数据字典中DEVICE表的status字段枚举值未包含“已出租”，而是“在库、已出租、维修中、异常、报废”。SRS正文中使用了“已出租”，但数据字典中status字段的说明列写的是“设备状态：在库、已出租、维修中、异常、报废”，两者一致，但“已出租”在数据字典中作为枚举值列出，而在FR-EQP-003中作为状态描述使用，存在术语使用上的轻微不一致。建议统一术语。 | 在数据字典的DEVICE.status字段说明中，明确列出所有枚举值，并与SRS正文中使用的状态术语完全一致。例如，将说明改为：'枚举值：在库、已出租、维修中、异常、报废'。 |
| 对话偏差 | 建议 | §3.1.2 FR-EQP-004 设备归还检测 | SRS描述“系统将录入的检测数据与设备出厂测试基准数据...进行自动偏差计算”，但涉众原话（BR-EQP-006）中招商业务员提到的是“系统会预先录入每台设备的出厂测试基准数据...当业务员在验收界面录入实际检测数值后，系统会自动与基准值进行偏差计算”。SRS未明确说明“出厂测试基准数据”由谁在何时录入。虽然FR-CFG-002提到运维工程师可配置检测清单，但未明确基准数据的录入职责。 | 在FR-EQP-004的前置条件或关联需求中，明确“出厂测试基准数据”的录入职责和时机。例如，增加一条关联需求：'运维工程师应在设备首次入库时，通过FR-CFG-002录入设备的出厂测试基准数据。' |
