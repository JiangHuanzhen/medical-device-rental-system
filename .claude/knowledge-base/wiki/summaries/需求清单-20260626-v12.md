| 需求来源 | raw/notes/运维工程师-20260626-1536-需求记录.md（运维工程师） |
| 需求陈述 | 「您提的这个问题非常关键，直接关系到设备出库前的质量控制和责任划分。从我们库房实际操作的角度来看，招商业务员建议的“科室主任→库房主管→质控部门”顺序，确实存在您担心的风险——**设备很可能在未完成质量检查的情况下就被出库**。具体来说，影响主要有以下几点：质控环节被后置，易出现“先出库、后检查”库房主管在第二环节签字时，主要核对的是设备实物、配件、档案是否齐全。但设备是否通过校准、状态是否合格，必须由质控部门最终确认。如果科室主任已经签了字，意味着使用方在主观上已认可可以出库，这时库房主管很难再以“质控未确认”为由卡住流程，只能被动走完。一旦质控最终发现设备需要校准或维修，设备已经到临床，可能会影响临床使用，甚至造成医疗风险。」 |
| 需求提炼 | 作为库房人员，我希望系统强制紧急出库审批按“库房主管→质控部门→科室主任”的顺序流转，以便避免因质控环节后置导致设备带病出库的风险。 |
| 优先级 | 高 |

| 需求来源 | raw/notes/运维工程师-20260626-1536-需求记录.md（运维工程师） |
| 需求陈述 | 「您提到的这个细节非常关键，从库房管理的专业角度和系统流程的严谨性出发，我的理解是：**弹窗本身不应直接对紧急出库“放行”，而是应该作为紧急出库申请的“入口”**。具体来说，系统在设备校准到期前7天弹出阻止常规出库的弹窗，这是为了确保设备合规性、避免带病流转。但为了应对突发救治等紧急场景，我们在设计流程时，建议在弹窗中增加一个“紧急出库申请”按钮。用户点击后，系统自动提交审批（比如科室负责人、质量管理部门或设备科领导审批），**审批通过后，系统才解除对该设备的出库锁定**。」 |
| 需求提炼 | 作为库房人员，我希望系统在设备校准到期前7天弹出的阻止出库弹窗中，增加一个“紧急出库申请”按钮，以便在紧急情况下，能通过审批流程解除锁定，实现快速响应。 |
| 优先级 | 高 |
