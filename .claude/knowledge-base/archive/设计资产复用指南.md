# 设计资产复用指南

## 资产包概览

本资产包来自「医疗器械租赁管理系统」项目，采用分层架构 + Spring Boot 3.x + Java 17 技术栈。五层结构如下：

```
.claude/
├── skills/                          ← 第四层：子技能层（5组 SKILL.md）
│   ├── completeness-check.md
│   ├── correctness-check.md
│   ├── consistency-check.md
│   ├── effectiveness-check.md
│   ├── change-impact-analysis.md
│   └── prompt-templates/            ← 第五层：提示词模板层
│       ├── arch-constraint-template.md
│       ├── quality-check-template.md
│       └── cia-template.md
│
├── knowledge-base/
│   └── wiki/
│       └── design/
│           ├── adr/                 ← 第三层：知识图谱层（ADR决策链）
│           ├── spec/                ← 第三层：知识图谱层（ASD/MDS/DTS）
│           ├── contracts/           ← 第二层：接口契约层（OpenAPI YAML）
│           └── reports/             ← 正向/逆向知识图谱 + 校验报告
│
project/
└── src/                             ← 第一层：代码层（v1 + v2）
```

## 如何在新项目中复用

### Step 1: 复制子技能和模板
```bash
cp -r .claude/skills/ <新项目>/.claude/skills/
```

### Step 2: 修改参数
每个 SKILL.md 顶部有参数表，将默认路径改为新项目的知识库路径。

### Step 3: 复用接口契约
```bash
cp wiki/design/contracts/openapi-*.yaml <新项目>/wiki/design/contracts/
```
接口契约是技术栈无关的——相同的业务域（如设备管理、租赁订单）可以直接参考字段定义和错误码设计。

### Step 4: 参考正向知识图谱
```bash
cp wiki/design/reports/正向知识图谱-*.json <新项目>/wiki/design/reports/
```
知识图谱捕捉了模块划分和依赖设计的**模式**——如「通知功能独立为服务，通过事件与其他模块解耦」这个模式在任何同类项目中都可复用。

### Step 5: 适配约束提示词模板
将 `arch-constraint-template.md` 中的 C-ARCH/C-MOD/C-CODE 约束替换为新项目的设计产物，保留固定前缀+变动后缀的缓存优化结构。

## 什么情况下不适用

| 资产 | 不适用场景 |
|------|-----------|
| SKILL.md 子技能 | 非分层架构项目需修改校验逻辑 |
| OpenAPI 契约 | 不同业务域需重新设计字段 |
| 约束提示词模板 | 事件驱动/微服务架构需替换前置约束 |
| 正向知识图谱 | 不同技术栈的模块组织方式不同 |

## 最低复用条件
- 新项目采用**分层架构**
- 新项目的设计产物存储在相同的四层知识库结构中
- 新项目使用 OpenAPI 3.0 标准定义接口
